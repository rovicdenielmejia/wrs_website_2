Phase 2 Set Up Database (Supabase)

We'll use Supabase as our PostgreSQL database provider. It's free and easy to set up.

1 Create Supabase Account
🌐
Where to go: Browser → supabase.com → Click "Start your project"
📸 Supabase Homepage
Screenshot: "Start your project" button highlighted at top right
2 Sign Up with Email

Enter your email address → Create a password → Click "Sign up" → Check your email for verification link

3 Create New Project
🔧
Where to go: After login → Click "New project"
Field What to Enter Example
Organization Your organization name (or create new) WRS Organization
Name Your project name wrs-database
Database Password Create a strong password (remember this!) MySecureP@ssw0rd123
Region Select nearest region Singapore (or closest)
📸 Supabase Create Project Form
Screenshot: Fill in name, password, and select region
4 Wait for Project to Initialize
⏳ This takes about 2-3 minutes. You'll see a loading screen while Supabase sets up your database.
5 Get Database Connection String
⚠️ Can't find it? Try these 3 ways:

Method 1: Direct Link (Easiest)

🔗
Copy this URL into your browser:
https://supabase.com/dashboard/project/YOUR-PROJECT/settings/database
Replace "YOUR-PROJECT" with your actual project name

Method 2: From Dashboard Settings

⚙️
Step-by-step:
1. Go to supabase.com and log in
2. Click on your project in the dashboard
3. Look for the gear icon ⚙️ on the left sidebar
4. Click "Database" in the settings menu
5. Scroll down until you see "Connection string"

Method 3: Alternative Connection Info

🔧
Look for these values instead:
• Host: db.xxxxxx.supabase.co (look for something like this)
• Port: 5432
• Database: postgres
• User: postgres
• Password: Your Supabase password (not your email password!)
📸 Supabase Settings → Database page
Screenshot: Look for the "Connection string" or "URI" section with a copy button
💡 What to copy: A string that looks like:
postgresql://postgres:YOUR-PASSWORD@db.xxxxxx.supabase.co:5432/postgres
⚠️ Important: Make sure to replace YOUR-PASSWORD in the string with your actual Supabase project password!
✅ Phase 2 Complete! Your Supabase database is ready. Keep the connection string safe - you'll need it soon!

Phase 3 Configure Local Project

Now let's set up the project on your computer.

1 Download/Clone the Project
📁
Where to save: Create a folder like H:\Projects\WRS
Command Prompt
cd H:\Projects\WRS git clone [YOUR-REPO-URL]

Or download as ZIP and extract to your folder.

2 Open Project in VS Code
💻
Where to run: VS Code → File → Open Folder → Select your project folder
3 Install Dependencies
VS Code Terminal (Ctrl+`)
npm install

This installs all required packages (might take 2-5 minutes)...

4 Create Environment File
📄
Where to create: In project root → Duplicate .env.example → Rename to .env.local

Or create a new file named .env.local

5 Edit .env.local with Real Values
✏️
Where to edit: VS Code → Open .env.local file 
# ===========================================
# EDIT THESE VALUES - REPLACE THE EXAMPLES
# ===========================================

# Database - Paste your Supabase connection string here
# (From Step 2 Phase 5)
DATABASE_URL="postgresql://postgres:YOUR-PASSWORD@db.xxxxxx.supabase.co:5432/postgres"

# Auth URL - Your production domain (we'll change this later)
NEXTAUTH_URL="http://localhost:3000"

# Generate secrets by running: openssl rand -base64 32
# Or use: https://minimax.dev/tools/password-generator
NEXTAUTH_SECRET="paste-your-generated-secret-here"
JWT_SECRET="paste-your-generated-secret-here"
⚠️ Important: Replace YOUR-PASSWORD with your actual Supabase password!
6 Generate Prisma Client
VS Code Terminal
npm run db:generate
7 Push Database Schema
VS Code Terminal
npm run db:push

This creates all tables in your Supabase database.

8 Start Development Server (Test)
VS Code Terminal
npm run dev

Now open your browser to: http://localhost:3000

🎉 Your local site is running! You should see the WRS website.
⚠️ Keep the terminal running! Press Ctrl+C to stop when done testing.
✅ Phase 3 Complete! Your local project is configured and working.

Where to add: Scroll down on same page → Find "Environment Variables" section

3 Push to GitHub

👁️

Phase 6 Configure Custom Domain

Now let's connect your own domain (like example.com).

1 Buy a Domain (if you don't have one)
🛒
Where to buy: GoDaddy, Namecheap, Cloudflare, or Google Domains

Search for your desired domain → Buy it (usually $10-20/year)

2 Add Domain to Vercel
🌍
Where to go: Vercel Dashboard → Your project → Settings → Click "Domains"
📸 Vercel Settings - Domains page
Screenshot: Domain input field

Enter your domain (e.g., workforcerecruitmentsolutions.com) → Click "Add"

3 Update Environment Variable
✏️
Where to update: Vercel Dashboard → Settings → Environment Variables

Find NEXTAUTH_URL → Change value to your domain: https://workforcerecruitmentsolutions.com → Save

4 Configure DNS Records (at your domain provider)
📡
Where to configure: Your domain provider's DNS settings
Type Name/Host Value
A @ (or leave blank) 76.76.21.21
CNAME www cname.vercel-dns.com
5 Wait for DNS Propagation
⏰ DNS changes can take 5 minutes to 48 hours. Check propagation at dnschecker.org
6 Verify SSL Certificate
🔒 Vercel automatically provisions SSL! You should see the lock icon 🔒 in your browser.
✅ Phase 6 Complete! Your site is now live at your custom domain!

🔧 Troubleshooting Common Issues

Build Failed

Check these:
  • All environment variables are set in Vercel
  • Node.js version is 18 or higher
  • DATABASE_URL is correct
📋
View build logs: Vercel Dashboard → Deployments → Click failed deployment → Scroll to view error messages

Database Connection Error

Check these:
  • DATABASE_URL format is correct
  • Password in connection string is correct
  • Database project is not paused (Supabase)
  • IP is not blocked

Site Not Loading

Check these:
  • Domain DNS is pointing correctly
  • SSL certificate is active (green lock)
  • Try clearing browser cache

Changes Not Showing

Make sure you:
  • Pushed code to GitHub
  • Waited for deployment to finish (2-5 min)
  • Hard refresh: Ctrl+Shift+R
